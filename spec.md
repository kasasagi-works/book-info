# 技術仕様書

## 概要

ISBNを入力するとopenBD APIから書籍情報を取得し、noteの読書記録記事に貼り付けられるマークダウンテキストを生成するWebサービス。

## システム構成

- **フロントエンド**：HTML + CSS + JavaScript（単一ファイル）
- **バックエンド**：不要（ブラウザから直接APIをfetch）
- **設置環境**：レンタルサーバー（静的ファイルとして配置）
- **CORS**：openBD APIはCORSヘッダーを返すため、ブラウザから直接リクエスト可能

## 使用API

**openBD API**  
エンドポイント：`https://api.openbd.jp/v1/get?isbn={ISBN}`

- 無料・認証不要
- レスポンス形式：JSON（配列）
- 該当書籍が存在しない場合：配列の要素が `null`

## 抽出データ仕様

レスポンスの `[0].summary` から以下の5項目を抽出する。

| フィールド名 | 項目名 | 型 | 備考 |
|---|---|---|---|
| `isbn` | ISBN | string | 13桁 |
| `title` | 書名 | string | |
| `publisher` | 出版者 | string | |
| `pubdate` | 出版年月 | string | `YYYYMMDD` または `YYYYMM` 形式。取得値をそのまま表示 |
| `author` | 著者名 | string | 著者が複数の場合は `／` 区切り |

項目名はopenBD JSONスキーマの `summary` オブジェクト内の `description` の値を使用。  
参照：https://api.openbd.jp/v1/schema?pretty

## 出力フォーマット

```
ISBN: **{isbn}**
書名: **{title}**
出版者: **{publisher}**
出版年月: **{pubdate}**
著者名: **{author}**
```

## ISBNの入力形式

ユーザーはハイフンあり・なしどちらでも入力可能。  
ハイフン・スペースを除去した上でAPIに渡す。

- 入力例：`978-4-7808-0204-7` → APIリクエスト：`9784780802047`
- バリデーション：除去後に10桁または13桁の数字であること

## エラーハンドリング

| ケース | 対応 |
|---|---|
| 未入力 | 「ISBNを入力してください。」を表示 |
| ISBN形式不正 | 桁数を含むエラーメッセージを表示 |
| 書籍が存在しない（`null`） | 「該当する書籍が見つかりませんでした。」を表示 |
| API通信エラー | 「APIへの接続に失敗しました。」を表示 |
| HTTPエラー | ステータスコードを含むエラーメッセージを表示 |
